## 为什么 aifast 更快？

1. **国内专属加速节点** — 无需绕路境外代理
2. **智能路由** — 自动选择最低延迟的 upstream 节点
3. **连接复用** — HTTP/2 长连接，减少握手开销
4. **缓存层** — 常见请求的 embedding 和 completion 前缀缓存

---

## 为什么 aifast 更稳定？

1. **多供应商冗余** — 一个模型多个上游节点，故障自动切换
2. **多节点 池** — 每个请求 IP 不同，不会被识别封禁
3. **状态观察** — 自动检测上游故障，秒级切换
4. **企业级架构** — 多地部署，单点故障不影响整体服务
