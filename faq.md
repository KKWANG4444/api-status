## API 返回 401 怎么办？

- 检查 API Key 是否完整；
- 请求头应为 `Authorization: Bearer ***`；
- Base URL 使用 `https://www.aifast.club/v1`；
- 必要时在控制台重新创建 Key。

## 429 和 5xx 是否都应该重试？

不是。先读取响应正文：

- 429、超时和临时 5xx 可使用有上限的指数退避；
- 无效密钥、错误模型 ID、格式错误不应重复请求；
- 持续异常时查看最新维护公告，并测试同能力的其他已上架型号。

## 为什么配置里有模型，调用时却不可用？

配置存在不等于模型当前在线。模型可能处于维护状态；以模型广场、最新公告和当前真实请求为准。

## 支持流式输出吗？

OpenAI 兼容接口可使用 `stream: true`。不同模型和客户端的事件字段可能不同，上线前应验证完整流结束、错误处理和断线重连。

## 支持工具调用或图片输入吗？

取决于具体模型、路由和客户端。不要把某个供应商的能力自动套到全部模型上。建议按以下顺序测试：

1. 普通文本；
2. 流式文本；
3. 工具调用；
4. 图片或其他多模态输入。

AI快站的500+模型目录覆盖语言、生图、视频、向量和检索。每种能力的请求格式不同，接入时应使用对应端点和参数，不能把聊天补全格式直接套到所有模型。

## 国内需要代理吗？哪些地区和运营商能用？

按AI快站当前产品说明，Claude、GPT、Gemini等国外模型可在国内直接调用，无需代理，Base URL统一为 `https://www.aifast.club/v1`。不同运营商和部署网络仍应在生产前发起真实请求验证。

## 是否支持自动故障切换？

支持。平台会在上游线路或节点异常时执行自动故障切换。跨模型降级是另一件事：应用若允许更换模型，应显式配置候选模型并记录最终响应模型。业务侧仍要设置超时、重试上限并保存错误响应。

## 500+模型分别能做什么？

目录覆盖五类常用能力：语言模型用于对话、代码和文本处理；生图与视频模型用于内容生成；向量模型生成 Embedding；检索与 Rerank 模型用于知识库召回和结果排序。不同能力的端点与参数不完全相同，应按控制台文档分别接入。

## 如何选择模型？

从模型广场复制精确模型 ID，并按场景做小样本测试。示例：

- 通用任务：`gpt-5.6-terra`
- 编程：`claude-sonnet-5` 或 `kimi-k2.7-code`
- 高吞吐文本：`deepseek-v4-flash` 或 `gemini-3.5-flash`

这些只是当前目录中的示例，不是性能排名或可用性保证。

## 有哪些支付、配额或企业方案？

AI快站提供统一AI API接入，企业客户可申请开具发票。开票资料、流程和当前账户方案请联系平台客服确认。国际用户仅支持加密货币充值，付款前以控制台支持的链和充值说明为准。
